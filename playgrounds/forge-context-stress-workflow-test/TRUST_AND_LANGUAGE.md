# Trust And Language Notes

The wording of the product matters almost as much as the feature set.

## Avoid these traps

- promising “seamless sync” before it is trustworthy
- describing the product as “smart” if it mostly applies deterministic local logic
- using judgmental wording around missed work or poor routines
- presenting optional portability as if it were a core guaranteed capability

## Prefer language like

- calm weekly recovery
- local-first planning
- interruption-friendly workflow
- clear next-step focus
- explicit portability boundaries

## Messaging risk

If the product says one thing and the architecture or rollout notes imply another, trust drops quickly.
