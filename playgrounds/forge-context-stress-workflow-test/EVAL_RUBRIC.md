# Evaluation Rubric

Use this rubric after running all five fresh sessions.

Score each criterion from `0` to `3`.

- `0`: poor or missing
- `1`: weak / inconsistent
- `2`: solid
- `3`: strong

Maximum score: `45`

## Session 1: Orientation And Product Synthesis

### 1. Orientation quality

Did the session orient early and appropriately for a continuity-heavy workspace?

- `0`: no meaningful orientation
- `1`: partial orientation, mostly file re-reading
- `2`: clear orientation with some Forge awareness
- `3`: early Forge-aware orientation with a focused start

### 2. Product synthesis quality

Did `PRODUCT_SPEC.md` become coherent rather than a dump of source notes?

### 3. Contradiction handling

Did the session make important tensions visible without trying to prematurely solve everything?

## Session 2: Architecture And Risks

### 4. Resume quality

Did the session resume from prior continuity instead of rediscovering the workspace from scratch?

### 5. Architecture usefulness

Did `ARCHITECTURE_PLAN.md` reflect the actual product and constraints instead of generic architecture talk?

### 6. Risk specificity

Did `RISK_REGISTER.md` capture real risks tied to this workspace, not generic filler?

## Session 3: Incident And Scope Change

### 7. Change detection

Did the session clearly recognize that the incident materially changed the direction of the work?

### 8. Explicit drift / issue handling

Was the change handled explicitly rather than being silently absorbed?

- `0`: silent drift
- `1`: acknowledged informally only
- `2`: explicit but weakly handled
- `3`: explicit and well handled in continuity state and deliverables

### 9. Scope correction quality

Did the deliverables become more trustworthy after the incident?

## Session 4: Review And Refinement

### 10. Review-feedback integration

Did the session use the review feedback to sharpen the package rather than just add more text?

### 11. Executive summary quality

Did `EXEC_SUMMARY.md` become short, useful, and aligned with the rest of the package?

### 12. Cross-document consistency

Did the package become tighter and more internally aligned after review?

## Session 5: Final Review And Closeout

### 13. Final resume quality

Did the final session resume from tracked continuity rather than redoing earlier synthesis?

### 14. Final state integrity

Did issues, drift, phase state, and handoff appear to match the real state of the workspace?

### 15. Closeout quality

Did the workspace feel genuinely finished rather than merely stopped?

## Cross-Session Forge Use

### 16. Appropriate Forge usage

Did the agent use the right amount of Forge over time?

- `0`: strong underuse or overuse
- `1`: mixed, with noticeable friction or noise
- `2`: mostly appropriate
- `3`: consistently appropriate and low-friction

### 17. Continuity leverage

Did Forge noticeably improve the agent’s ability to carry momentum and intent across sessions?

### 18. Handoff quality over time

Did session handoffs make later sessions stronger?

### 19. State density discipline

Did the agent avoid turning Forge into noisy bookkeeping?

### 20. WorkingView leverage

Did the behavior suggest that `workingView` guidance was actually helping decisions?

## Score interpretation

- `38-45`: Forge is clearly adding major value in this class of workflow.
- `30-37`: Forge is helpful but still has noticeable friction or consistency gaps.
- `20-29`: Forge is partially useful but not yet reliably improving multi-session execution.
- `0-19`: Forge is not yet pulling its weight for this kind of stress workflow.

## Qualitative questions

After scoring, answer these:

1. Where did Forge help most?
2. Where did the agent still fall back to rediscovery?
3. Did the agent over-write state anywhere?
4. Did the agent miss any moment that should have triggered explicit issue or drift handling?
5. What is the single highest-value product improvement suggested by this run?
